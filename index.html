PascalCase란?

PascalCase(또는 UpperCamelCase)는 모든 단어의 첫 글자를 대문자로 시작하며, 단어와 단어 사이에 공백이나 구분선 없이 직접 붙여 쓰는 네이밍 컨벤션입니다.

주요 특징 및 쓰임새

  • 객체 지향 프로그래밍(OOP)에서 클래스(Class) 및 인터페이스(Interface) 이름 정의에 주로 사용됩니다.
  • 대부분의 백엔드 언어(Java, C#, C++)와 모던 스택(TypeScript, Python, Go 등)의 구조체/클래스 공식 네이밍 권장 규칙입니다.
  • 첫 글자도 대문자로 쓰기 때문에 소문자로 시작하는 camelCase와 명확하게 구분됩니다.

권장 적용 분야

언어/환경 대상 요소
Java / C# 클래스, 인터페이스
TypeScript / React 컴포넌트 이름, 타입/인터페이스
Python 클래스명 (PEP 8 규칙)

// TypeScript Class & Interface Example
interface UserProfileData {

    return true;

camelCase란?

camelCase는 첫 단어는 항상 소문자로 시작하고, 두 번째 단어부터는 첫 글자를 대문자로 시작하는 네이밍 기법입니다. 단어들의 높낮이가 낙타의 등에 있는 혹과 비슷하다고 하여 붙여진 이름입니다.

주요 특징 및 쓰임새

  • 변수(Variable) 및 함수/메서드(Function/Method) 이름을 명명할 때 가장 대중적으로 사용됩니다.
  • JavaScript, TypeScript, Java 등 프론트엔드와 범용 웹 백엔드 시스템의 핵심 기본 코딩 관례입니다.
  • JSON 데이터 포맷의 키(Key) 값을 정의할 때도 대다수의 모던 API가 camelCase를 준수합니다.

권장 적용 분야

언어/환경 대상 요소
JavaScript / TS 일반 변수, 함수명, 오브젝트 키
Java / C# 로컬 변수, 메서드명(Java)
REST API JSON Payload 응답 키

// JavaScript Variables & Functions

let isUserAccountActive = true;

  const calculatedDiscount = basePrice * (discountRate / 100);
  return calculatedDiscount;
}

// JSON Payload Example
const userProfileResponse = {

  }

snake_case란?

snake_case는 모든 글자를 소문자로 표기하며, 단어와 단어 사이를 언더바/밑줄(_) 기호로 연결하는 네이밍 컨벤션입니다. 옆으로 기어가는 뱀의 형상을 빗대어 이름이 정해졌습니다.

주요 특징 및 쓰임새

  • Python 언어의 공식 PEP 8 스타일 가이드에서 함수와 변수명에 적극적으로 권장하는 방식입니다.
  • 데이터베이스(RDB)의 테이블명 및 열(Column)명을 작명할 때 전통적으로 가장 널리 쓰이는 표준입니다.
  • 대문자로만 이루어진 UPPER_SNAKE_CASE는 소스 코드 내의 전역 상수(Constant)를 정의할 때 약속처럼 사용됩니다.

권장 적용 분야

언어/환경 대상 요소
Python / Rust 변수, 함수, 모듈명
SQL / Database 테이블명, 컬럼명 (e.g. user_id)
전체 언어 공통 전역 상수 정의 (UPPER_SNAKE)

# Python Variable and Function Style

kebab-case란?

kebab-case(또는 spinal-case)는 모든 단어를 소문자로 표기하고 단어 사이를 대시/하이픈(-)으로 연결하는 텍스트 포맷입니다. 꼬치구이(Kebab)에 재료가 꿰어져 있는 구조에서 유래되었습니다.

주요 특징 및 쓰임새

  • HTML 속성(Attribute) 및 CSS 클래스/아이디 명명 시 표준 관례입니다. (예: class="btn-primary")
  • URL 경로(Routing Path) 작성 시 가독성을 해치지 않으며 SEO(검색 최적화)에 가장 적합한 이름 표기법으로 권장됩니다.
  • 프로그래밍 언어 소스 코드 내에서는 연산자 뺄셈(-)과 기호가 겹치기 때문에 일반적인 변수명으로는 쓸 수 없습니다.

권장 적용 분야

언어/환경 대상 요소
HTML / CSS 클래스명, ID명, data-* 속성
Web URLs RESTful URL 슬러그 (slug)
Package / Files npm 패키지 이름, 폴더/파일 에셋명

Antigravity 2.0 적용법

복사한 프롬프트 및 변수를 Antigravity 2.0에 등록하여 지속 사용하려면 아래 두 방법 중 선택하세요.

  1. Rule(규칙)로 상시 지정:
    프로젝트 루트에 .agents/AGENTS.md를 생성하거나, PC 전역 설정인 C:\Users\redof\.gemini\config\AGENTS.md 파일 끝에 아래 양식으로 위 프롬프트를 추가합니다.
    # CSS Guidelines [복사한 AI 프롬프트 삽입]
  2. Skill(스킬)로 개별 지정:
    프로젝트 내 .agents/skills/skin-custom/SKILL.md 경로로 새 폴더 및 마크다운을 만듭니다. 앞부분에 아래 메타데이터(Frontmatter)를 지정하고 본문에 코드를 삽입하세요.
    --- name: design-skin description: CSS 테마 디자인 스킨 가이드 --- # CSS Skin Variables [복사한 CSS 테마 변수 삽입]
